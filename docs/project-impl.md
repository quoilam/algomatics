# 技术要求
- 所有模块互相独立, 逻辑、业务严格强解耦, 各模块与子模块间责任边界明显, 任务清晰, 便于团队开发与维护
- 我已有TAVILY_API_KEY  OPENROUTER_BASE_URL OPENROUTER_API_KEY 等环境变量, 可以直接调用TavilyAPI和OpenRouterAPI实现检索和代码生成的功能
- 这里的openrouterapi是openai兼容的, 你可以直接使用openai、openai-agents这些包来用, 不需要自己再实现chatCompletion接口
- 不需要考虑工程化、项目部署、安全等问题, 只需要实现基本的功能逻辑即可, 这个项目的主要目的是验证设计思路, 以及实现一个基本的可用的系统, 后续可以在这个基础上进行优化和改进
- 不需要考虑ci test privacy security等问题


交互要求
整个流程都以网页端对话的形式呈现, 需要如下组件
- 对话框: 展示用户输入的提示词和示例图片, 以及系统返回的结果, 包括生成的图片、算法代码、评估结果等等
  - 反馈窗口: 内嵌在对话框内, 由controller触发展示, 用户可以对生成的图片和算法代码进行反馈, 选择Accept或 Reject with suggestions, 反馈结果会被controller接收并处理, 进行下一轮的迭代优化
- 状态展示图: 实时展示agent的调用情况, 包括正在调用哪个agent, 调用的参数是什么, 返回的结果是什么, 以及不同agent之间的调用关系等等, 以图形化的方式展示出来, 让用户能够清晰地看到整个流程的执行情况, 以及不同agent之间的协作情况
  - 可以基于mermaid或者其他图形化工具来实现, 只需要能够清晰地展示agent的调用情况和流程即可, 不需要过于复杂和精美的设计
- 底部输入框: 用于用户输入提示词和示例图片, 用户可以输入文本提示词, 也可以上传示例图片
- 聊天历史记录窗口, 用于展示历史聊天记录
