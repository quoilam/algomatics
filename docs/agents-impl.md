# Agent编排
--- 反向引用: 见 [设计文档](design.md) ---

大致要有如下几个subagent 

- 检索Agent: 联网检索场景相关内容, 例如实现思路、评价指标等等
  - 调用TavilyAPI实现, 并且能够将结果结构化输出
  - 能够做本地搜索记录缓存, 避免重复检索 (简单实现)
  
- 代码生成Agent: 通过一些外部知识(例如检索到的结果)或大模型本身的知识, 结合本地安装的图像处理库(由开发者提前安装好, 如opencv pil等, 由prompt或tool/skill description给出), 生成对应的代码
  - 需要有系统上下文 (用户设备 硬件配置 操作系统等), 这部分初始化时通过prompt给出, 生成代码时也需要考虑到这些因素
  - 需要有环境上下文, 开发者提前安装好常用的图像处理工具库, 生成代码时需要考虑到这些工具库的使用方法和限制, 例如Opencv pil等
  - 需要有独立的对话上下文, 生成代码时需要考虑到之前的对话内容, 之前生成的代码内容等等, 才能够满足微调、小patch的需求

- 评估Agent: 调用多模态大模型以及结合外部知识, 以及其他上下文数据, 对生成的图片质量做出评价
  
- 执行Agent: 用于执行算法代码生成图片, 总结前序操作链路, 以及执行过程中遇到的错误, 将结果返回给控制器
  - 不需要考虑沙箱、隔离等安全问题, 只需要能够执行代码, 并且将结果返回给控制器即可
